# Workspace Report - Báo Cáo

Dùng thư mục này cho report khoảng 15 trang và bằng chứng hỗ trợ.

Cấu trúc khuyến nghị:

1. Vấn đề và phạm vi.
2. Requirement và kịch bản nghiệm thu cuối.
3. Kiến trúc và trade-off.
4. Domain model và data flow.
5. Tích hợp Facebook.
6. Tích hợp email.
7. Kafka, Redis và reliability.
8. Security và privacy.
9. Testing và verification.
10. Hướng dẫn demo và screenshots.
11. Giới hạn và phương án production-grade.
12. Bài học rút ra.

Mọi claim trong report phải được hỗ trợ bằng implementation, tests, logs,
screenshots hoặc limitation được document rõ.
